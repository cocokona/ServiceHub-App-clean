基于 `docs/OFFLINE_SYNC_DESIGN.md` v1.0 完整构建了离线缓存与数据同步系统，按三层架构：

### Phase 1: 基础设施层（8 个文件）
- `src/services/cacheConfig.ts` — 缓存配置注册中心（6 种数据类型的 TTL/粒度/淘汰策略）
- `src/services/storage.service.ts` — AsyncStorage 抽象层（LRU 淘汰、容量管理 200MB 配额）
- `src/services/cache.service.ts` — 三级缓存（L1 Memory → L2 AsyncStorage → L3 Supabase, cache-aside read + write-through）
- `src/services/syncConfig.ts` — 同步规则注册中心（增量字段、冲突策略 last_write_wins/merge/optimistic_lock）
- `src/services/network.service.ts` — NetInfo 网络监测（200ms 防抖、弱网检测 30s 内 3 次断开）
- `src/services/syncQueue.service.ts` — 离线队列（持久化、批量同步、指数退避重试 max 3 次）
- `src/services/sync.service.ts` — 同步编排器（增量拉取 + 队列推送 + 冲突解决）
- `src/services/index.ts` — 统一导出

### Phase 2: 通用组件层（4 个文件）
- `src/components/OfflineBanner.tsx` — 离线横幅（灰色 #F0F0F0 + spring 动画）
- `src/components/SyncIndicator.tsx` — 同步指示器（banner/bottom/dot 三种模式）
- `src/components/LoadingSkeleton.tsx` — 骨架屏（card/list/detail/inline 四种变体 + shimmer 动画）
- `src/components/index.ts` — 统一导出

### Phase 3: Hook 层（2 个文件）
- `src/hooks/useNetworkStatus.ts` — 网络状态 Hook
- `src/hooks/useOfflineData.ts` — 离线优先数据获取 Hook（内置三层缓存、离线降级、后台同步）

### 依赖
- 安装 `@react-native-community/netinfo`

### 接入方式
新页面只需调用 `useOfflineData({ key: 'jobs', fetchFn })`，自动获得离线/在线数据获取能力。
现有页面渐进迁移时只需替换数据获取方式为 useOfflineData。

## 数据与逻辑分离重构

将所有硬编码数据从组件/配置文件提取到独立 JSON 数据文件，通过运行时 loader 统一加载。

### 新增数据层
- `src/data/files/` — 18 个 JSON 数据文件（按域分类）
- `src/data/loader.ts` — 运行时加载器（safeGet/safeArray 安全访问 + resolveAvatar avatarKey→URL 间接引用 + 诊断 API）
- `src/data/index.ts` — Barrel 导出
- `src/data/constants.ts` — 向后兼容 shim（deprecated, re-export from loader）

### 迁移的源文件（18 个）
- 13 个屏幕组件：CustomerHome/ServiceDetails/ScheduleDetails/Checkout/Tracking/SupportChat/TechnicianDashboard/JobDetails/ActiveService/ServiceCompletion/SelectRole/AppNavigator/AuthScreen
- 3 个配置/服务：api/client.ts, services/cacheConfig.ts, services/syncConfig.ts
- 1 个类型声明：nativewind-env.d.ts（添加 `declare module '*.css'`）
- 1 个向后兼容：constants.ts

### 关键设计
- avatarKey 间接引用：mock 数据中用短键名（如 `avatarKey: "tech1"`），loader 加载时 `resolveAvatar()` 解析为完整 URL
- safeGet/safeArray：数据缺失返回 fallback + 推入 loadWarnings 数组
- cacheConfig.ts 和 syncConfig.ts 直接 import JSON，不经过 loader（因为是服务层配置，不涉及头像解析）
- 修复 nativewind-env.d.ts TS2882 错误（添加 `declare module '*.css'`）
