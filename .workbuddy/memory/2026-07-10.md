## 修复 RCTFatal：NSLocation*UsageDescription 缺失
- **现象**：原生崩溃 `EXBaseLocationRequester getPermissions` → `This app is missing usage descriptions...`。app.json 的 expo-location 插件只在 `expo prebuild` 时把 key 写进 Info.plist，而提交的 `ios/servicehubmobile/Info.plist` 从未含该 key（在装 expo-location 之前就 build 过）。
- **修复**：直接在 `ios/servicehubmobile/Info.plist` 加 `<key>NSLocationWhenInUseUsageDescription</key>` + 描述串（与 app.json 插件文案一致），`plutil -lint` 校验通过。保留 app.json 插件以便将来 clean prebuild 仍一致注入。
- **关键**：仅改 Info.plist 不够，必须**重新编译原生 app**（`expo run:ios` 或 Xcode 跑 `ios/servicehubmobile.xcworkspace`，或 `eas build` 重装 dev build）。JS 热更新不会改变 native plist。
- **提醒给用户的坑**：expo config plugin 对原生 plist/entitlements 的修改都需重建原生二进制，不是 `expo start` 就能生效。
