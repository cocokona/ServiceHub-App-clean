# 2026-07-08 工作日志

## order_in_progress 表找不到 + 数据库连接配置修复

### 问题
后端报错 `Could not find the table public.order_in_progress in the schema cache`

### 根因
1. 迁移 `00004_order_in_progress.sql` 缺少 `NOTIFY pgrst, 'reload schema'`，PostgREST schema cache 未刷新
2. `src/lib/supabase.ts` 硬编码了 Supabase URL 和 anon key
3. `app-config.json` 残留幻影 API 地址 `192.168.1.100:3000`

### 修复
- 迁移末尾添加 `NOTIFY pgrst, 'reload schema'`
- 新建 `src/config/env.ts` 集中化环境配置模块（requireEnv 校验）
- 新建 `.env` / `.env.example` 环境变量文件
- `supabase.ts` 改为从 config/env.ts 导入，移除所有硬编码凭据
- `app-config.json` 修正为 `localhost:3000`
- `loader.ts` 的 `apiBaseUrl` 优先读 `EXPO_PUBLIC_API_BASE_URL`
- `.gitignore` 确保 `.env` 被忽略
- 表名一致性验证：10 张表全部匹配，0 处不一致
- `tsc --noEmit` 零错误

### 架构变更
- 新增 `src/config/` 目录，集中管理所有环境敏感配置
- 环境变量前缀统一为 `EXPO_PUBLIC_`（Expo build-time 注入）
